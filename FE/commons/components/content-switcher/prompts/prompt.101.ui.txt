핵심요구사항) MCP를 연동하여 피그마의 디자인을 그대로 소스코드로 구현할 것.

핵심요구사항) 다음 조건을 모두 만족하는 완전한 variant 시스템을 구현할 것.
            - variant: 'active' | 'inActive'

핵심요구사항) content-switcher은 탭 메뉴와 동일한 기능일 것.
      1. 메뉴 아이템 (상수 정의)
        - 메뉴 목록은 외부 API가 아닌 컴포넌트 내부에 상수로 고정(Hardcoded)합니다.
        - 목록: `["스튜디오", "드레스", "메이크업", "웨딩홀"]`
      2. 선택 로직
        - 탭 메뉴와 동일하게 동작하며, 한 번에 하나의 아이템만 선택 가능할 것 (Single Selection).
        - 다중 선택(Multi-select)은 불가능할 것.
        - 클릭 시 해당 아이템의 Variant가 `active`로 변경되고, 나머지는 `inActive`가 될 것

핵심요구사항) 시각적 디자인 및 레이아웃을 다음과 같이 구현할 것.
      1. 색상 시스템
        - 배경색: #F8F8F8
        - 글자색: #71727A (active/inactive 모두 동일)
        - active 배경색: #fff
        - divider 색상: #C5C6CC
      2. 크기 및 레이아웃
        - 전체 컨테이너 너비: 345px
        - 각 탭 영역: divider를 고려하여 동적으로 계산된 크기 (약 83.5px)
        - 탭 높이: 31px
        - 컨테이너 패딩: 4px (좌우)
        - cornerRadius: 16px (컨테이너), 12px (탭)
      3. Divider 기능
        - 각 탭 사이에 divider 표시 (#C5C6CC, 1px × 12px)
        - active 탭 양쪽에는 divider 표시하지 않음 (opacity로 공간 유지)
        - 탭 클릭 시 레이아웃 움직임 없도록 공간 고정
