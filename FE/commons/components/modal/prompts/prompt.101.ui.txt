        1) 모달에 사용될 <Button/> 공통컴포넌트 원본을 수정하지 말고, 아래의 props만 활용할 것.

                - disabled
        2) 모달에 사용될 <Dropdown/> 공통컴포넌트 원본을 수정하지 말고, 아래의 props만 활용할 것.

                - options: {value, label}
                - onChange
                - placeholder?
                - disabled?
        3) 모달에 사용될 <Input/> 공통컴포넌트 원본을 수정하지 말고, 아래의 props만 활용할 것.

                - size?

핵심요구사항) 모달은 하나의 통짜 컴포넌트가 아니라, 아래 요소들을 조합하여 렌더링하는 구조일 것.
        1.  컨테이너 (Container)
        - `ModalProvider`와 함께 사용되므로, 자체적인 Backdrop(검은 배경)을 절대 포함하지 말 것.
        - 배경색: Figma 참조 (모든 모달 동일)
        - Width: Figma 참조 (모든 모달 동일)
        - Padding: Vertical 21px, Horizontal 30px (Figma 값 재확인 후 적용)
        - Border Radius: Figma 참조

        2.  구성 요소 슬롯 (Slots & Props)
        - `title`: (string, optional) 상단 타이틀
        - `description`: (string, optional) 타이틀 하단 설명 텍스트
        - `children`: (ReactNode) Input, Dropdown, Message 등 가변적인 컨텐츠가 들어갈 영역
        - `buttons`: (Object) 하단 고정 버튼 설정 (아래 하단 버튼 영역(Footer) 항목 참조)

핵심요구사항) 모달 내부 `children`으로 들어갈 공통 컴포넌트들은 아래 규칙을 엄격히 준수할 것. (기존 컴포넌트 수정 금지)
        1.  Button (하단 액션용)
        - `variant`: 'filled' (확인 버튼) | 'outlined' (취소 버튼)
        - `size`: 'small' 고정
        - `icon`: false
        - `disabled`: 부모로부터 전달받은 상태 적용
        2.  Dropdown (본문용)
        - `values`, `options`({value, label}), `onChange` 사용
        - `placeholder`, `disabled` 속성 활용
        3.  Input (본문용)
        - `label`, `size`, `disabled`, `placeholder`, `value` 속성 활용

핵심요구사항) 하단 버튼 영역(Footer)은 고정된 레이아웃을 가질 것.
        1.  버튼 구성: 최대 2개의 버튼(좌: 취소/닫기, 우: 확인/저장)이 배치 - 내용 변경 가능
        2.  상태 연동 (중요)
        - 오른쪽(확인) 버튼의 `disabled` 상태는 모달 내부의 Input이나 Dropdown의 선택 여부에 따라 제어되어야 합니다.
        - 구현 방식: Modal 컴포넌트 자체는 비즈니스 로직(입력값 검증)을 알 수 없으므로, `buttons` prop을 통해 `disabled` 상태값과 `onPress` 핸들러를 외부에서 주입받는 구조로 구현
        - 예: `buttons={{ right: { label: '확인', onPress: handleOk, disabled: !isFormValid } }}`

핵심요구사항) 시각적 디자인 및 레이아웃을 다음과 같이 구현할 것.
        - `styles.ts`에 스타일을 정의하되, 두 컴포넌트용 스타일을 명확히 구분할 것.
        - `color.ts`와 `typography.ts`에 정의된 상수를 최우선으로 사용하고, 없는 경우에만 위 Hex 코드를 사용할 것.
        - 인라인 스타일은 절대 사용하지 말고, 모든 스타일은 `styles.ts`에 정의할 것.
        - 하드코딩된 색상값(hex/rgb/hsl)은 절대 사용하지 말고, 반드시 `color.ts`의 토큰을 사용할 것.

핵심요구사항) 모달 그림자 효과를 다음과 같이 구현할 것.
        - 피그마 Drop shadow 설정을 정확히 반영할 것.
        - Position: X:0, Y:0
        - Blur: 20
        - Spread: -3.48
        - Color: #800C3A (10% opacity)
        - `colors.modal.shadow` 토큰을 사용할 것 (없으면 `color.ts`에 추가).
        - iOS: shadowColor, shadowOffset, shadowOpacity, shadowRadius 사용
        - Android: elevation 사용
