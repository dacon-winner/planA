핵심요구사항) 다음 조건을 모두 만족하는 완전한 variant 시스템을 구현할 것.
            - variant: 'active' | 'inActive'

핵심요구사항) filter은 아래와 같이 동작할 것.
      1. 메뉴 아이템 (상수 정의)
        - 메뉴 목록은 외부 API가 아닌 컴포넌트 내부에 상수로 고정(Hardcoded)할 것.
        - 목록: `["전체","스튜디오", "드레스", "메이크업", "웨딩홀"]`
      2. 선택 로직
        - 사용자 클릭은 토글(Toggle) 방식으로 동작하며, "전체" 버튼과 "개별" 버튼 간의 연동 로직은 아래 규칙을 엄격히 따를 것.
          Scenario 1: '전체' 버튼 클릭 시
          - IF 현재 '전체'가 OFF 상태:
            -> '전체'를 ON으로 변경하고, 나머지 모든 아이템도 ON으로 변경.
          - IF 현재 '전체'가 ON 상태:
            -> '전체'를 OFF로 변경하고, 나머지 모든 아이템도 OFF로 변경.

          Scenario 2: '개별' 아이템(스/드/메/웨) 클릭 시
          - 해당 아이템의 상태를 토글(ON <-> OFF).
          - 후처리 검증:
            - IF 나머지 모든 개별 아이템이 ON 상태가 되었다면 -> '전체' 버튼도 자동으로 ON.
            - IF 개별 아이템 중 하나라도 OFF 상태가 되었다면 -> '전체' 버튼은 자동으로 OFF
